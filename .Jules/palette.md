## 2026-03-18 - Canvas Game HUD Accessibility and Visual Sync
**Learning:** In canvas-based WebGL/Three.js games, critical game state updates (like the upcoming building tier) are often trapped in the canvas, making them inaccessible to screen readers. Creating a synchronized DOM overlay using `aria-live="polite"` and `aria-atomic="true"` solves this a11y issue natively. Furthermore, synchronizing the DOM text color with the 3D placement cursor's material color creates a seamless cognitive link for all users between the HUD and the 3D environment without overwhelming the screen.
**Action:** When working with canvas/WebGL games, always check if crucial interaction state is exposed to the DOM with ARIA live regions, and look for opportunities to visually sync DOM HUD elements with 3D cursors/indicators using shared color palettes.
