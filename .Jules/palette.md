## 2024-05-14 - Bridging WebGL Game State with Accessible DOM Overlays
**Learning:** In Canvas/WebGL games, critical game state updates displayed in DOM overlays (like the upcoming building tier) can be made accessible to screen readers by using `aria-live="polite"` and `aria-atomic="true"`. This ensures screen readers announce changes dynamically without interrupting the user's flow. Exposing critical visual states like the cursor color to the DOM overlay syncing logic ensures consistency between the visual WebGL canvas and the accessible DOM.
**Action:** When implementing DOM overlays for canvas games, identify critical game state variables (like next items, scores, or alerts) and use `aria-live` attributes to sync these updates to screen readers seamlessly. Sync any related visual canvas state variables within the same DOM update function to guarantee UI consistency.
